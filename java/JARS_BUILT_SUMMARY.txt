================================================================================
DEPLOYMENT OPTIONS
================================================================================

Option 1: Standalone JAR (Simple)
  java -jar tron-cli-jar-with-dependencies.jar <command>
  java -jar opentron-java-backend-0.1.0.jar

Option 2: Docker (Recommended for Backend)
  docker build -t opentron-backend:0.1.0 backend/
  docker run -p 8000:8000 -e ENGINE_HOST=http://ollama:11434 opentron-backend:0.1.0

Option 3: Installation (Linux/Mac)
  cp backend/target/opentron-java-backend-0.1.0.jar /usr/local/bin/
  cp cli/target/tron-cli-jar-with-dependencies.jar /usr/local/bin/tron
  chmod +x /usr/local/bin/tron
  
  Then use directly:
    tron ask "What is Docker?"

Option 4: systemd Service (Linux)
  [Unit]
  Description=OpenTron Backend
  After=network.target
  
  [Service]
  Type=simple
  ExecStart=/usr/bin/java -jar /opt/opentron/opentron-java-backend-0.1.0.jar
  Restart=on-failure
  
  [Install]
  WantedBy=multi-user.target

CONFIGURATION
================================================================================

Backend Configuration (environment variables):

  TRON_PORT              - Server port (default: 8000)
  ENGINE_HOST            - LLM engine URL (default: http://localhost:11434)
  ENGINE_API_KEY         - API key for remote engines (optional)
  SERVER_ADDRESS         - Server bind address (default: 0.0.0.0)
  OPENTRON_API_KEY       - API key for backend clients (optional)

Example:
  export ENGINE_HOST=http://localhost:11434
  export TRON_PORT=8080
  java -jar opentron-java-backend-0.1.0.jar

  telemetry.db           - Usage metrics
  models.json            - Model registry
