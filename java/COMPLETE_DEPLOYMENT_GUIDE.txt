================================================================================
ENVIRONMENT CONFIGURATION
================================================================================

Backend Configuration (Environment Variables):

1. Port Configuration:
   set TRON_PORT=8000
   (default: 8000)

2. LLM Engine Connection:
   set ENGINE_HOST=http://localhost:11434
   (default: http://localhost:11434 for Ollama)

3. Engine API Key (if required):
   set ENGINE_API_KEY=your-api-key
   (optional, for remote engines)

4. Server Bind Address:
   set SERVER_ADDRESS=0.0.0.0
   (default: 0.0.0.0 - all interfaces)

5. API Key for Backend Clients:
   set OPENTRON_API_KEY=your-api-key
   (optional, for authentication)

Example - Set all variables before starting:

PowerShell:
$env:TRON_PORT="8000"
$env:ENGINE_HOST="http://localhost:11434"
$env:ENGINE_API_KEY="optional-key"
java -jar opentron-java-backend-0.1.0.jar

CMD:
set TRON_PORT=8000
set ENGINE_HOST=http://localhost:11434
set ENGINE_API_KEY=optional-key
java -jar opentron-java-backend-0.1.0.jar

  Delete: %USERPROFILE%\.OpenTron\

TROUBLESHOOTING
================================================================================

Backend won't start:
  ✓ Check port 8000 is free: netstat -ano | findstr :8000
  ✓ Check Java 17+: java -version
  ✓ Check file exists: dir C:\...backend\target\*.jar
  ✓ Check disk space: dir C:\

Backend returns 404:
  ✓ Verify LLM engine (Ollama) is NOT required for /v1/savings, /v1/models
  ✓ These are handled by ForwardingController fallback
  ✓ Check ForwardingController.java was rebuilt

CLI command not found:
  ✓ Use full JAR name: tron-cli-jar-with-dependencies.jar
  ✓ Don't use: opentron-java-cli-0.1.0.jar (requires classpath)
  ✓ Verify file exists

CLI can't connect to backend:
  ✓ Backend must be running on port 8000
  ✓ Test: curl http://localhost:8000/health
  ✓ Check firewall (localhost should be fine)

Memory not persisting:
  ✓ Check ~/.OpenTron/ directory is writable
  ✓ Check disk space available
  ✓ Run: java -jar tron-cli-jar-with-dependencies.jar init

Out of memory:
  ✓ Increase heap: java -Xmx2g -jar opentron-java-backend-0.1.0.jar
  ✓ Check system memory: wmic OS get TotalVisibleMemorySize

PRODUCTION DEPLOYMENT
================================================================================

For Windows Production:

1. Create deployment directory:
   mkdir C:\OpenTron-Prod
   mkdir C:\OpenTron-Prod\backend
   mkdir C:\OpenTron-Prod\cli

2. Copy JARs:
   copy backend\target\opentron-java-backend-0.1.0.jar C:\OpenTron-Prod\backend\
   copy cli\target\tron-cli-jar-with-dependencies.jar C:\OpenTron-Prod\cli\

3. Create start script (start-backend.cmd):
   @echo off
   cd C:\OpenTron-Prod\backend
   set TRON_PORT=8000
   set ENGINE_HOST=http://localhost:11434
   java -jar opentron-java-backend-0.1.0.jar

4. Register as Windows Service (optional):
   Install NSSM: https://nssm.cc/
   nssm install OpenTronBackend "java" "-jar C:\OpenTron-Prod\backend\opentron-java-backend-0.1.0.jar"
   nssm set OpenTronBackend AppDirectory C:\OpenTron-Prod\backend
   nssm start OpenTronBackend

5. Verify running:
   curl http://localhost:8000/v1/savings
   nssm status OpenTronBackend

DOCKER DEPLOYMENT (Optional)

3. Test:
   curl http://localhost:8000/v1/savings

   docker-compose up -d
