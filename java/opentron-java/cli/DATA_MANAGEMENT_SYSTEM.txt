COMPONENT 1: DataManager.java (13.3 KB)
================================================================================

Core file and configuration management utility.

KEY FEATURES:
  ✓ Configuration file loading/saving (TOML format)
  ✓ Directory initialization
  ✓ JSON file I/O
  ✓ CSV import/export
  ✓ Log file management
  ✓ File size formatting
  ✓ Backup creation
  ✓ File deletion (recursive)

PUBLIC METHODS:
  - initializeDirectories()          Create ~/.OpenTron structure
  - loadConfig()                     Load configuration from file
  - saveConfig(Map)                  Persist configuration
  - getDefaultConfig()               Get baseline config
  - saveJson(Path, Object)           Write JSON data
  - loadJson(Path, Class)            Read JSON data
  - appendLog(String, String)        Add log entry with timestamp
  - readLogTail(String, int)         Read last N log lines
  - listFiles(String)                Directory listing
  - getFileSize(String)              Get file size in bytes
  - formatFileSize(long)             Human-readable size (B, KB, MB, etc.)
  - exportCsv(String, List)          Export data as CSV
  - importCsv(String)                Import CSV data
  - backup(String)                   Create timestamped backup
  - getLastModified(String)          Get file modification time
  - delete(String)                   Delete file/directory
  - getConfigDir()                   Get ~/.OpenTron path

STORAGE LOCATIONS:
  - Config:        ~/.OpenTron/config.toml
  - Persona:       ~/.OpenTron/SOUL.md, MEMORY.md, USER.md
  - Logs:          ~/.OpenTron/daemon.log, memory.log, vault.log
  - Backups:       ~/.OpenTron/*.backup_YYYYMMDD_HHMMSS

USAGE EXAMPLES:
  DataManager.initializeDirectories();
  Map<String, Object> config = DataManager.loadConfig();
  config.put("model", "qwen2.5:7b");
  DataManager.saveConfig(config);
  DataManager.appendLog("~/.OpenTron/daemon.log", "Server started");

COMPONENT 2: MemoryStore.java (6.7 KB)
================================================================================

Persistent knowledge and memory management system.

KEY FEATURES:
  ✓ In-memory JSON-backed database
  ✓ Full-text search across content, tags, categories
  ✓ Category-based organization
  ✓ Tag support for entries
  ✓ Automatic timestamp tracking
  ✓ UUID-based entry IDs
  ✓ JSON/CSV export formats
  ✓ Persistent storage (~/.OpenTron/memory.db)

PUBLIC METHODS:
  - addEntry(String, String, List)   Add memory entry with category/tags
  - search(String)                   Full-text search entries
  - getByCategory(String)            Filter entries by category
  - deleteEntry(String)              Remove entry by ID
  - clear()                          Delete all entries
  - getCount()                       Get total entries
  - getAll()                         Retrieve all entries
  - exportJson()                     Export as JSON
  - exportCsv()                      Export as CSV
  - save()                           Persist to storage

DATA STRUCTURE:
  MemoryEntry {
    String id              - Unique identifier (UUID)
    String content         - Full text content
    long timestamp         - Creation time (ms since epoch)
    String category        - Organization category (e.g., "general")
    List<String> tags      - Searchable tags
  }

USAGE EXAMPLES:
  MemoryStore store = new MemoryStore();
  store.addEntry("Important info", "work", Arrays.asList("urgent", "task"));
  List<MemoryEntry> results = store.search("task");
  for (MemoryEntry entry : results) {
    System.out.println(entry);  // [timestamp] content (tags: urgent, task)
  }
  store.exportCsv();  // Returns CSV format
  store.save();       // Persist to disk

COMMAND INTEGRATION:
  tron memory list              - List all entries
  tron memory search <query>    - Search entries
  tron memory add <text>        - Add entry
  tron memory delete <id>       - Delete entry
  tron memory export -f csv     - Export as CSV
  tron memory clear             - Delete all entries

COMPONENT 3: VaultStore.java (7.1 KB)
================================================================================

Encrypted secrets and credentials management.

KEY FEATURES:
  ✓ AES-like encryption (Base64 + XOR for demo, upgrade to real crypto)
  ✓ Persistent encrypted storage
  ✓ Secret type tracking
  ✓ Access time recording
  ✓ Creation timestamp
  ✓ Metadata retrieval without decryption
  ✓ Bulk operations (list, clear)

PUBLIC METHODS:
  - setSecret(String, String, String)  Store secret with type
  - getSecret(String)                  Retrieve and decrypt secret
  - deleteSecret(String)               Remove secret
  - listSecrets()                      Get all secret keys
  - hasSecret(String)                  Check existence
  - getMetadata(String)                Get metadata without decryption
  - clear()                            Delete all secrets
  - getCount()                         Total secrets
  - exportVault()                      Export encrypted format

DATA STRUCTURE:
  Secret {
    String value           - Encrypted secret value
    String type            - Secret type (e.g., "api_key", "password")
    long created           - Creation timestamp
    long accessed          - Last access time
  }

  SecretMetadata {
    String key             - Secret identifier
    String type            - Secret type
    long created           - Creation timestamp
    long accessed          - Last access time
  }

ENCRYPTION:
  Current: Base64 + XOR (demo)
  Production: Use Java Cryptography Extension (JCE) for AES-256

USAGE EXAMPLES:
  VaultStore vault = new VaultStore();
  vault.setSecret("github-token", "ghp_xxxxx", "api_key");
  String token = vault.getSecret("github-token");
  vault.deleteSecret("github-token");
  System.out.println(vault.getCount());  // Total secrets

COMMAND INTEGRATION:
  tron vault list              - List secret keys
  tron vault set <key> <val>   - Store secret
  tron vault get <key>         - Retrieve secret
  tron vault delete <key>      - Remove secret
  tron vault clear             - Delete all secrets
  tron vault export            - Export vault contents

STORAGE:
  ~/.OpenTron/vault.json       - Encrypted vault file
  ~/.OpenTron/vault.log        - Access log

COMPONENT 4: TelemetryStore.java (10.2 KB)
================================================================================

Usage metrics and performance monitoring system.

KEY FEATURES:
  ✓ Event recording with full details
  ✓ Query by time range, command, model, engine
  ✓ Automatic statistics calculation
  ✓ CSV/JSON export
  ✓ Configurable enable/disable
  ✓ Bounded history (max 10000 events)
  ✓ Summary statistics generation

PUBLIC METHODS:
  - recordEvent(String, String, String, long, int, boolean)  Log event
  - getSummary()                 Get aggregate statistics
  - queryByTimeRange(long, long) Filter by timestamp range
  - queryByCommand(String)       Filter by command name
  - queryByModel(String)         Filter by model ID
  - clear()                      Delete all events
  - exportCsv()                  Export as CSV
  - exportJson()                 Export as JSON
  - setEnabled(boolean)          Enable/disable collection
  - getEventCount()              Total events
  - getRecent(int)               Last N events
  - save()                       Persist to disk

DATA STRUCTURE:
  TelemetryEvent {
    String id              - Unique event ID
    long timestamp         - Event timestamp (ms)
    String command         - Command executed
    String model           - Model used
    String engine          - Engine backend
    long duration_ms       - Execution time
    int tokens             - Tokens generated
    boolean success        - Success flag
  }

  TelemetrySummary {
    int total_queries
    int successful_queries
    int failed_queries
    double average_latency_ms
    int total_tokens
    double throughput_tokens_per_sec
    String top_model
    String top_engine
  }

USAGE EXAMPLES:
  TelemetryStore telemetry = new TelemetryStore();
  telemetry.recordEvent("ask", "qwen2.5:7b", "ollama", 245, 150, true);
  TelemetrySummary stats = telemetry.getSummary();
  System.out.println(stats);  // Summary output
  List<TelemetryEvent> recent = telemetry.getRecent(10);

COMMAND INTEGRATION:
  tron telemetry status        - Show status
  tron telemetry enable        - Enable collection
  tron telemetry disable       - Disable collection
  tron telemetry summary       - Show statistics
  tron telemetry recent        - Show recent events
  tron telemetry export -f csv - Export data
  tron telemetry clear         - Clear database

STORAGE:
  ~/.OpenTron/telemetry.db     - Event database (JSON)

================================================================================
SECURITY CONSIDERATIONS
================================================================================

VAULT ENCRYPTION:
  Current:  Base64 + XOR (demo implementation)
  Recommended: AES-256-CBC with key derivation

SECRETS IN LOGS:
  - Vault logs record access, not values
  - Telemetry excludes sensitive data
  - Configuration file should not contain secrets

BACKUP STRATEGY:
  - Automatic timestamped backups
  - Manual backup() method available
  - Use DataManager.backup(filePath)

ACCESS CONTROL:
  - Use UNIX file permissions on ~/.OpenTron/
  - Restrict to user: chmod 700 ~/.OpenTron/
  - Regular security audits recommended

================================================================================
PERFORMANCE CHARACTERISTICS
================================================================================

MEMORY STORE:
  - Bounded at 10,000 entries
  - Search: O(n) linear scan
  - Add/Delete: O(n) with list operations
  - Recommended: Archive old entries monthly

VAULT STORE:
  - Fast lookup: O(1) HashMap
  - Encryption/decryption overhead minimal
  - Safe for 100s of secrets

TELEMETRY STORE:
  - Max 10,000 events (auto-truncation)
  - Search: O(n) with filtering
  - Export: O(n) CSV generation
  - Consider external analytics for high volume

MODEL REGISTRY:
  - Fast lookup: O(1) HashMap
  - Search: O(n) substring matching
  - Supports 1000s of models efficiently
