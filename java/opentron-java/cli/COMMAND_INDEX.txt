COMMON PATTERNS ACROSS ALL COMMANDS
====================================

Argument Parsing:
  - Long form: --flag value
  - Short form: -f value
  - Boolean flags: --flag
  - Variable arguments

Output Format:
  println()          - Display output (respects --quiet)
  printlnErr()       - Error output
  debug()            - Debug output (--verbose only)
  errorExit()        - Error and exit

Error Handling:
  - try-catch blocks
  - Graceful error messages
  - Exit with proper codes
  - Stacktrace on --verbose

COMMON SUBCOMMANDS PATTERN
============================

Most commands follow this pattern:

tron <command> <subcommand> [options]

Examples:
  tron model list
  tron model pull qwen2.5:7b
  tron scheduler schedule task_name --cron "0 7 * * *"
  tron vault set key_name secret_value
